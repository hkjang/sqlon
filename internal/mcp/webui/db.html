1. 데이터베이스 엔진

엔진에 따라 드라이버와 접속 방식, SQL 방언이 결정됩니다. 잘못 선택하면 연결할 수 없습니다.

2. 운영 컨텍스트

플릿 화면의 분류·위험도·담당 조직에 사용됩니다. 운영 환경과 중요도는 정확히 지정하세요.

운영(production) 프로파일은 평문 비밀번호를 저장할 수 없습니다. env: 또는 file: Secret 참조를 사용하세요.

3. 기본 접속 정보

PostgreSQL 접속 정보를 입력하세요.

형식: host:port/database 또는 postgres://host:port/database
Docker에서는 env: 변수를 컨테이너 environment로 전달하고, file: 경로는 컨테이너에 마운트해야 합니다.

4. Oracle 연결·라이선스 정책

Pack 사용 여부는 운영자가 보유 계약을 확인하여 선언하며 기본값은 모두 비활성입니다.

Oracle 모니터링 계정에는 SYS를 사용할 수 없습니다. Wallet·비밀번호는 감사 로그에 기록되지 않습니다.
저장 후 실제 DB에 Ping하여 실패 단계와 해결 방법을 표시합니다.

🔍 감상 인덱스 성능 시뮬레이터

실제 테이블에 인덱스를 생성하지 않고, 감상 인덱스가 SQL 성능(Cost)에 미치는 영향을 시뮬레이텘합니다. (PostgreSQL의 경우 hypopg 확장 기능 지원 시 실측 계획 반영)
