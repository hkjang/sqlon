# sqlon 온보딩 가이드

**sqlon**은 PostgreSQL·MySQL·MariaDB에 대한 **메타데이터 기반 NL2SQL MCP 서버**입니다. 자연어 질문을 카탈로그(테이블·컬럼·관계·지표·용어집)에 근거해 안전한 read-only SQL로 바꾸고 실행합니다. 이 문서는 처음 쓰는 분을 위한 빠른 안내입니다.

## 1. 핵심 개념

- **카탈로그**: 물리/논리 모델, 관계(조인 그래프), 용어집, 지표, 코드사전, 오버라이드를 컴파일한 인메모리 메타데이터. NL2SQL의 근거입니다.
- **DB 프로파일**: 실제 접속 정보(postgres/mysql/mariadb). 쿼리는 항상 **읽기 전용 세션**으로 실행됩니다.
- **원칙**: 물리 구조는 자동 수집, **업무 의미(논리명·지표 등)는 승인 후에만** 카탈로그에 반영합니다.

## 2. 5분 시작

1. **DB · 쿼리** 화면에서 DB 프로파일을 등록하고 연결을 테스트합니다.
2. **질의** 화면에서 자연어로 질문합니다. 내부적으로 `prepare_sql_context`가 테이블·컬럼·조인·시간조건·검증 힌트를 한 번에 묶어 줍니다.
3. 재질문이 나오면 답한 뒤 다시 질문 → 스켈레톤의 `/* SLOT */`을 채워 SQL 완성.
4. **검증 → 실행계획 → 실행** 순으로 진행하면 가드레일(행 제한·타임아웃·PII 마스킹·실행계획 게이트)이 자동 적용됩니다.

## 3. 메타데이터 관리 워크플로

- **자동 수집**: `run_metadata_sync`로 원천 DB의 물리 모델을 스냅숏으로 수집하고, `apply_metadata_sync`로 카탈로그에 반영합니다(기존 설명 보존, 삭제는 폐기 후보).
- **의미 보강**: `suggest_semantic_metadata`·`suggest_model_candidates`가 논리명·설명·코드사전·지표·관계 **후보**를 생성합니다.
- **검토 승인**: **메타 검토** 화면에서 후보를 승인/반려하고, 승인분을 overrides/metrics/relations에 반영합니다.
- **품질 관리**: **메타 품질** 화면에서 7차원 점수·등급·릴리스 게이트를 확인합니다.

## 4. OpenMetadata 연동 (선택)

전사 카탈로그 **OpenMetadata**와 양방향 연동합니다. **OpenMetadata** 화면에서 연결을 설정하고, 큐레이션된 설명·PII·용어집을 빈 필드에만 import하거나, sqlon 설명을 export하고, 두 카탈로그의 불일치(drift)를 대조합니다.

## 5. 프로파일별 카탈로그 (멀티 DB)

**프로파일 카탈로그** 화면에서 각 DB 프로파일마다 독립 카탈로그 워크스페이스를 관리합니다.

1. **라이브 DB로 구축/갱신** 또는 **전체 구축**으로 물리 모델을 워크스페이스에 수집합니다.
2. `import_openmetadata_to_profile`로 그 DB의 업무 메타데이터를 수급합니다.
3. 데이터셋 JSON을 조회·편집(검증·백업·롤백)합니다.
4. **활성 카탈로그로 전환**(무재기동)하거나, 요청 단위로 `profile`을 지정해 여러 DB를 동시에 질의합니다.

## 6. 운영·보안

- **감사 로그**: 모든 도구 호출·관리 작업이 해시 체인 감사 로그에 남아 변조를 탐지합니다.
- **메트릭**: `/metrics`(Prometheus)에서 도구 호출량·품질 점수 등을 노출합니다.
- **스케줄러**: `-sync-interval`로 주기적 sync/import/다이제스트를 서버 내장 실행합니다.
- **권한**: 인증(메타 DB) 모드에서 관리 작업은 admin 역할이 필요합니다.
